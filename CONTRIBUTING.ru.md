# Руководство по внесению вклада

Спасибо за ваш интерес к внесению вклада в Awesome ChatGPT Prompts! 

## Как внести вклад

Самый простой способ внести вклад — через **[prompts.chat](https://prompts.chat)**:

1. Посетите [prompts.chat](https://prompts.chat)
2. Войдите с вашим аккаунтом GitHub
3. Создайте и отправьте ваш промпт
4. Ваш вклад автоматически синхронизируется с этим репозиторием

Ваше имя пользователя GitHub будет указано как участник, и вы появитесь в списке участников репозитория.

## Руководство по промптам

При создании нового промпта:

- **Тестируйте свой промпт** — Убедитесь, что он генерирует нужные результаты и может быть использован другими
- **Будьте описательны** — Дайте вашему промпту четкое, лаконичное название
- **Будьте оригинальны** — Не отправляйте дубликаты существующих промптов
- **Будьте уместны** — Контент должен быть подходящим для всех аудиторий

## Прямые вклады

Для исправления багов, улучшения документации или других вкладов, не связанных с промптами:

1. Сделайте форк репозитория
2. Создайте ветку для ваших изменений
3. Отправьте pull request с описательным заголовком и объяснением

## Формат CSV промптов

Если вы вносите вклад напрямую в `prompts.csv`, следуйте этому формату:

| Столбец | Описание |
|--------|-------------|
| `act` | Название промпта (без префикса "Действуй как") |
| `prompt` | Полный текст промпта |
| `for_devs` | `TRUE` или `FALSE` — это для разработчиков? |
| `type` | `TEXT` или `STRUCTURED` |
| `contributor` | Ваше имя пользователя GitHub |

**Правила CSV:**
- Экранируйте кавычки двойным цитированием (`""`)
- Никаких пробелов после запятых внутри цитируемых полей
- Используйте ваше имя пользователя GitHub точно так, как оно отображается

## Политика вопросов и проблем

Откройте issue, если у вас есть вопросы о внесении вклада. 

**Важно:** Этот репозиторий строго для AI-промптов. 
- **Не** публикуйте рекламу, запросы поддержки стороннего ПО (например, Norton Antivirus, проблемы с оплатой) или поддельные номера технической поддержки. 
- Любые не по теме issues будут немедленно закрыты, а пользователь, разместивший их, будет сообщен в GitHub за спам и вредоносную активность.
